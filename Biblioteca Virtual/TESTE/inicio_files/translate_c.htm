Login and Registration Form with HTML5 and CSS3 Faça login e Formulário de Inscrição com HTML5 e CSS3

A tutorial on how to create a switching login and registration form with HTML5 and CSS3. Um tutorial sobre como criar um login de comutação e formulário de inscrição com HTML5 e CSS3.



View demo Download source Veja a demonstração fonte Transferir

In this tutorial we are going to create two HTML5 forms that will switch between login and registration using the CSS3 pseudo class :target . Neste tutorial, vamos criar dois formulários HTML5 que vai alternar entre o login eo registro usando a classe CSS3 pseudo : alvo . We will style it using CSS3 and an icon font. Vamos denominá-lo usando CSS3 e uma fonte de ícones. The idea behind this demo is to show the user the login form and provide a link to “switch” to the registration form. A idéia por trás deste demo é para mostrar ao usuário o formulário de login e fornecer um link para "switch" para o formulário de inscrição.

Note that this is for demo purpose only, it will only work in browser supporting the :target pseudo class, and you should not use this code on a live website without providing solid fallback. We've added some HTML5 goodness here and used some of the new inputs. Nós adicionamos alguns bens HTML5 aqui e usou algumas das novas entradas. The input type=password automatically hides what the user is typing and replaces it with dots (depending on browser). O tipo de entrada = senha oculta automaticamente que o usuário está digitando e substitui-lo com pontos (dependendo do browser). The input type=email enables the browser to check if what the user entered has the format of a valid email address. O tipo de entrada = email permite que o navegador deve verificar se o que o usuário digitou tem o formato de um endereço de e-mail válido. We've also used the require=required attribute; Também usei a exigir = atributo obrigatório; browsers that support this attribute will not let the user submit the form until this field is filled, no JavaScript required. navegadores que suportam esse atributo não vai permitir que o usuário enviar o formulário até que este campo é preenchido, sem JavaScript necessário.
The autocomplete=on attribute will prefill values based on earlier user input. O autocomplete = on atributo prefill valores com base na entrada do usuário antes. We also used some nice placeholders for the inputs that will show some guiding value when the input is not filled. Nós também usamos algumas substituições agradáveis ​​para as entradas que vão mostrar algum valor orientador quando a entrada não foi preenchido.

Now the two tricky parts. Agora, as duas partes complicadas. You might have noticed the two <a href> links at the top of the form. Você deve ter notado os dois links href> <a na parte superior do formulário. This is a little trick that will make our form behave nicely when playing with anchors, so that it won't “jump” on long pages when we click on the switching link and trigger the :target pseudo-class. Este é um pequeno truque que vai fazer o nosso formulário se comportar muito bem quando se joga com âncoras, para que ele não vai "saltar" das páginas longas quando eu clico no link de comutação e acionar o: target pseudo-classe.

The second little trick is related to the use of the icon font. O segundo truque pouco está relacionado com a utilização do ícone da fonte. We will be using a data-attribute to display the icons. Nós estaremos usando um atributo de dados para exibir os ícones. By setting data-icon=”icon_character” with the according character in the HTML we will just need one CSS attribute selector to style all the icons. Ao definir-icon data = "icon_character" com o personagem de acordo no HTML nós só precisa de um seletor de atributo CSS para estilizar todos os ícones. Read more about this technique on 24 Ways: Displaying Icons with Fonts and Data- Attributes . Leia mais sobre esta técnica em 24 Maneiras: mostrar ícones com Fontes e Dados- Atributos .

Creating the switching animation Criando a animação comutação

The first thing to do is to hide the second form by setting the opacity to 0: A primeira coisa a fazer é esconder a segunda forma, definindo a opacidade para 0: 

#register{ #register {	
	z-index: 21; z-index: 21;
	opacity: 0; opacidade: 0;
} }

Remember that our login form had a z-index of 22? Lembre-se que o nosso formulário de login tinha um z-index de 22? We will give the second form a z-index of 21, to put it “under” the login form. Nós daremos a segunda forma um z-index de 21, para colocá-lo "em" o formulário de login.

And now the really good part: switching the forms using the :target pseudo class. E agora a parte realmente boa: mudar as formas, usando o: classe pseudo-alvo. What you really have to understand about :target, is that we will use anchors to make the transition. O que você realmente tem que entender sobre: ​​destino, é que nós vamos usar âncoras para fazer a transição. The normal behavior of an anchor link, is to jump to the target in the page. O comportamento normal de um link de âncora, é saltar para o destino na página. But we don't want to jump anywhere, we only want to switch the forms. Mas nós não queremos saltar em qualquer lugar, nós só queremos mudar as formas. And here comes our trick using the two links at the top of the page. E aqui vem o nosso truque usando os dois links no topo da página. Instead of directly linking to the second form, and risking getting a “jumping” effect, we actually put the two links at the top of the page and give them display: none . Em vez de diretamente ligando para a segunda forma, e arriscar a obtenção de um efeito de "saltar", nós realmente colocar os dois links no topo da página e dar-lhes display: none. This will avoid any page jump. Isso irá evitar qualquer página salto. Credit where credit's due: I found this trick on CSS3 create (in French). Crédito onde o crédito devido: Encontrei esse truque em CSS3 criar (em francês).

	animation-name: fadeInLeft; animação-name: fadeInLeft;

} }
